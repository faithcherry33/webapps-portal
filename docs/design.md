# 학습용 웹앱 포털 Design Guide

문서 버전: v0.1  
작성 목적: 학습용 웹앱 포털의 화면 구성, 디자인 원칙, 컴포넌트 스타일, 반응형 기준을 정의한다.

---

## 1. 디자인 목표

### 1.1 핵심 목표
학습용 웹앱 포털은 학생이 앱을 빠르게 찾고 실행할 수 있는 앱 런처형 화면이어야 한다.  
교사에게는 앱을 추가하고 관리할 수 있는 단순한 관리자 화면을 제공한다.

### 1.2 디자인 키워드
- 쉽다
- 밝다
- 정돈되어 있다
- 앱을 고르기 쉽다
- 초등학생도 혼자 사용할 수 있다
- 교사는 빠르게 추가할 수 있다

### 1.3 피해야 할 방향
- 과도한 대시보드형 화면
- 작은 글씨와 작은 버튼
- 과목별 메뉴가 너무 많은 구조
- 설명이 긴 카드
- 관리자 기능이 학생 화면에서 너무 눈에 띄는 구조
- 색상만으로 상태를 구분하는 디자인

---

## 2. 전체 디자인 콘셉트

### 2.1 콘셉트
“우리 반 학습 앱 서랍장”

학생이 포털에 들어오면 여러 앱이 정돈된 서랍이나 런처처럼 보이고, 필요한 앱을 골라 누르면 바로 활동을 시작하는 느낌을 준다.

### 2.2 메인 사용 장면
- 수업 시간에 교사가 “포털에서 #독서 태그 눌러요.”라고 안내한다.
- 학생은 포털에서 카테고리 또는 태그를 눌러 앱을 찾는다.
- 학생은 앱 카드의 썸네일과 제목을 보고 활동을 시작한다.
- 교사는 새 앱을 만든 뒤 관리자 모드에서 링크와 썸네일을 등록한다.

## 4. 레이아웃 원칙

### 4.1 전체 폭
- 최대 콘텐츠 폭은 1120px 내외로 설정한다.
- 큰 화면에서는 중앙 정렬한다.
- 모바일에서는 좌우 여백 16px을 기본으로 한다.

### 4.2 여백
- 화면 바깥 여백: 16~24px
- 섹션 간 여백: 24~32px
- 카드 내부 여백: 16px
- 버튼 내부 여백: 세로 10~12px, 가로 14~18px

### 4.3 카드 그리드
- 데스크톱: 3~4열
- 태블릿: 2열
- 모바일: 1열
- 카드 간격: 16~20px

### 4.4 시각적 우선순위
1. 포털 제목
2. 검색창
3. 카테고리
4. 자주 쓰는 태그
5. 앱 카드
6. 관리자 버튼

관리자 버튼은 필요하지만 학생 화면의 중심 요소가 아니므로 작고 차분하게 배치한다.

## 5. 색상 가이드

### 5.1 기본 색상 방향
밝은 배경 위에 흰색 카드가 올라가는 구조를 추천한다.  
학생용이므로 차갑고 딱딱한 업무 도구 느낌보다는 부드럽고 밝은 느낌을 우선한다.

### 5.2 추천 색상 토큰

```css
:root {
  --color-bg: #F7F9FC;
  --color-surface: #FFFFFF;
  --color-text: #1F2937;
  --color-muted: #6B7280;
  --color-border: #E5E7EB;
  --color-primary: #4F7DF3;
  --color-primary-soft: #EAF0FF;
  --color-success: #2F9E44;
  --color-warning: #F59F00;
  --color-danger: #E03131;
}
```

### 5.3 카테고리 색상
카테고리는 색상만으로 구분하지 않고 텍스트 라벨을 함께 표시한다.

```css
--category-class: #EAF0FF;      /* 수업 */
--category-study: #EAF7EE;      /* 학습 */
--category-play: #FFF4D6;       /* 놀이 */
--category-manage: #F3E8FF;     /* 학급운영 */
```

### 5.4 색상 사용 규칙
- 주요 버튼은 primary 색상을 사용한다.
- 선택된 카테고리는 배경색과 테두리, 굵은 글씨로 구분한다.
- 태그는 연한 배경의 칩 형태로 표현한다.
- 오류는 빨간색과 문구를 함께 사용한다.
- 성공은 초록색과 문구를 함께 사용한다.

## 6. 타이포그래피

### 6.1 글꼴
기본 시스템 글꼴을 사용한다.

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

한글 환경에서는 다음 순서를 고려할 수 있다.

```css
font-family: "Pretendard", "Noto Sans KR", system-ui, sans-serif;
```

단, 외부 폰트 로딩이 부담되면 시스템 기본 글꼴만 사용해도 충분하다.

### 6.2 글자 크기
- 페이지 제목: 28~32px
- 섹션 제목: 18~22px
- 앱 카드 제목: 18px
- 본문 설명: 14~15px
- 태그/라벨: 12~13px
- 버튼: 15~16px

### 6.3 문장 스타일
- 학생에게 보이는 문장은 짧게 쓴다.
- 버튼 문구는 동사형으로 쓴다.
- 예: 시작하기, 태그 해제, 저장하기, 다시 입력하기

## 7. 컴포넌트 디자인

### 7.1 Header

#### 구성
- 포털 이름
- 안내 문구
- 검색창
- 관리자 버튼

#### 디자인
- 상단에 넓은 여백을 둔다.
- 제목은 크고 명확하게 표시한다.
- 관리자 버튼은 오른쪽에 작게 배치한다.

#### 예시 문구
```text
우리반 학습 포털
필요한 학습 앱을 골라 시작하세요.
```

---

### 7.2 SearchBar

#### 목적
앱 제목, 설명, 태그로 빠르게 앱을 찾게 한다.

#### 디자인
- 너비는 가능한 넓게 둔다.
- 돋보기 아이콘을 사용할 수 있다.
- placeholder 문구는 구체적으로 쓴다.

#### Placeholder
```text
앱 이름이나 #태그를 검색하세요
```

#### 상태
- 기본
- 입력 중
- 검색 결과 없음
- 검색어 삭제 가능

---

### 7.3 CategoryTabs

#### 목적
전체, 수업, 학습, 놀이, 학급운영으로 앱을 분류한다.

#### 디자인
- 버튼형 탭으로 표시한다.
- 모바일에서는 가로 스크롤을 허용한다.
- 선택된 탭은 배경색, 테두리, 굵은 글씨로 강조한다.

#### 탭 목록
```text
전체 / 수업 / 학습 / 놀이 / 학급운영
```

---

### 7.4 TagFilter

#### 목적
같은 태그가 달린 앱을 모아 보게 한다.

#### 디자인
- 칩 형태로 표시한다.
- `#` 기호를 항상 붙여 보여준다.
- 선택된 태그는 primary 색상 계열로 강조한다.
- 태그가 많으면 2줄까지만 보이고, 더보기 버튼을 제공할 수 있다.

#### 예시
```text
#독서 #퀴즈 #경제 #빙고 #질문 #문해력
```

---

### 7.5 AppCard

#### 목적
학생이 앱의 성격을 보고 바로 실행하게 한다.

#### 구성
- 썸네일
- 카테고리 라벨
- 앱 제목
- 설명
- 태그
- 시작하기 버튼

#### 디자인
- 카드 배경은 흰색
- 모서리는 16~20px 둥글게
- 그림자는 아주 약하게
- 썸네일은 16:9 비율
- 버튼은 카드 하단에 고정

#### 카드 예시
```text
[썸네일]
수업
독서 퀴즈 풀기
돋보기, 탐정, 거울 문제를 풀며 책 내용을 정리해요.
#독서 #퀴즈 #문해력
[시작하기]
```

#### Hover / Focus
- 마우스를 올리면 카드가 살짝 떠오르는 느낌
- 키보드 포커스 시 명확한 테두리 표시
- 터치 기기에서는 hover 효과에 의존하지 않는다.

---

### 7.6 AdminCodeModal

#### 목적
관리자 코드 입력을 통해 앱 관리 기능에 접근한다.

#### 디자인
- 모달 또는 별도 페이지로 구현할 수 있다.
- 학생이 실수로 눌러도 혼란스럽지 않게 단순하게 만든다.
- 비밀번호 입력창 형태를 사용한다.

#### 오류 메시지
```text
관리자 코드가 올바르지 않습니다.
다시 입력해 주세요.
```

---

### 7.7 AdminAppForm

#### 목적
교사가 새 웹앱을 추가한다.

#### 필드
- 앱 제목
- 앱 링크
- 카테고리
- 태그
- 썸네일 이미지
- 간단 설명
- 공개 여부

#### 디자인 원칙
- 필수 항목에는 표시를 붙인다.
- 저장 전 썸네일 미리보기를 제공한다.
- URL 오류는 입력칸 바로 아래에 표시한다.
- 저장 성공 후 성공 메시지를 표시한다.

#### 성공 메시지
```text
웹앱이 추가되었습니다.
포털 목록에서 확인할 수 있습니다.
```

---

### 7.8 ThumbnailUploader

#### 목적
앱 카드에 표시될 썸네일 이미지를 업로드한다.

#### 권장 규격
- 비율: 16:9
- 최소 크기: 640 x 360px
- 권장 크기: 1280 x 720px
- 파일 형식: jpg, jpeg, png, webp
- 파일 크기: 2MB 이하 권장

#### 기본 썸네일
썸네일이 없는 앱에는 기본 이미지를 표시한다.

기본 이미지 방향:
- 연한 배경
- 앱 아이콘 또는 단순 도형
- 앱 제목 첫 글자 또는 카테고리 아이콘

## 9. 반응형 기준

### 9.1 데스크톱
- 화면 폭: 1024px 이상
- 카드 그리드: 3~4열
- 검색창과 관리자 버튼을 같은 줄에 배치 가능
- 카테고리와 태그는 여러 줄 표시 가능

### 9.2 태블릿
- 화면 폭: 768~1023px
- 카드 그리드: 2열
- 검색창은 한 줄 전체 사용
- 관리자 버튼은 검색창 아래나 오른쪽에 배치

### 9.3 모바일
- 화면 폭: 767px 이하
- 카드 그리드: 1열
- 카테고리 탭은 가로 스크롤
- 태그 영역은 접기/더보기 가능
- 버튼은 터치하기 쉽게 크게 표시

## 10. 접근성 기준

### 10.1 기본 기준
- 버튼은 키보드로 이동 가능해야 한다.
- 포커스 상태가 눈에 보여야 한다.
- 색상만으로 선택 상태를 표현하지 않는다.
- 썸네일 이미지에는 대체 텍스트를 제공한다.
- 텍스트 대비를 충분히 확보한다.

### 10.2 버튼 크기
- 최소 터치 영역은 44 x 44px 이상을 권장한다.
- 초등학생 사용을 고려해 주요 버튼은 이보다 여유 있게 만든다.

### 10.3 문구
- 오류 메시지는 원인과 해결 방법을 함께 제시한다.
- 예: `앱 링크는 https://로 시작해야 합니다.`

## 12. 아이콘과 썸네일 방향

### 12.1 아이콘 사용
카테고리별 기본 아이콘을 둘 수 있다.

- 수업: 🧑‍🏫 또는 📚
- 학습: ✏️ 또는 📝
- 놀이: 🎲 또는 🧩
- 학급운영: 🏫 또는 💬

### 12.2 썸네일 디자인
썸네일은 앱의 내용을 설명하는 작은 표지 역할을 한다.  
교사가 직접 이미지를 올릴 수 있으므로, 시스템은 이미지를 예쁘게 보이게 하는 프레임을 제공한다.

### 12.3 기본 썸네일 규칙
- 썸네일이 없는 경우 카테고리별 기본 배경을 표시한다.
- 앱 제목의 첫 글자를 크게 표시하는 방식도 가능하다.
- 기본 썸네일에도 `thumbnailAlt`를 제공한다.

## 16. 최종 디자인 방향 요약

이 포털은 예쁜 홈페이지보다 “수업 시간에 바로 쓰기 쉬운 앱 런처”가 되어야 한다.  
학생에게는 큰 카드, 쉬운 문구, 명확한 시작 버튼을 제공하고, 교사에게는 최소 입력만으로 앱을 등록할 수 있는 관리 화면을 제공한다.

카테고리는 `전체 / 수업 / 학습 / 놀이 / 학급운영`으로 단순하게 유지하고, 세부 분류는 `#태그`로 해결한다.  
이 구조가 앱이 늘어났을 때도 가장 유연하다.
